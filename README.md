# Credit Card Customer Segmentation

![Python](https://img.shields.io/badge/Python-3.8%2B-blue?logo=python&logoColor=white)
![Scikit-learn](https://img.shields.io/badge/Scikit--learn-1.3-orange?logo=scikit-learn&logoColor=white)
![Streamlit](https://img.shields.io/badge/Streamlit-1.x-red?logo=streamlit&logoColor=white)

An end-to-end unsupervised machine learning project that segments credit card customers into four distinct behavioural groups using clustering algorithms. The project covers exploratory data analysis, feature engineering, comparison of five clustering algorithms, dimensionality reduction with PCA, cluster profiling, and a Streamlit web app for real-time customer classification.

---

## Dataset

| Property       | Detail                                      |
|----------------|---------------------------------------------|
| Source         | Credit card behavioural dataset             |
| Rows           | ~8,950                                      |
| Features       | 18 (17 after dropping CUST_ID)              |
| Target         | None (unsupervised)                         |
| Null columns   | MINIMUM_PAYMENTS, CREDIT_LIMIT              |

Key features include BALANCE, PURCHASES, CASH_ADVANCE, CREDIT_LIMIT, PAYMENTS, MINIMUM_PAYMENTS, PRC_FULL_PAYMENT, TENURE, and various frequency/transaction count columns.

---

## Methodology

1. **Exploratory Data Analysis** — null heatmaps, distribution plots, correlation analysis
2. **Feature Engineering** — median imputation for null columns, drop CUST_ID
3. **Preprocessing** — StandardScaler normalisation
4. **Algorithm Comparison** — five clustering methods evaluated on Silhouette Score
5. **Model Selection** — KMeans with K=4 chosen based on highest silhouette score and interpretability
6. **Dimensionality Reduction** — PCA to 2D for cluster visualisation
7. **Cluster Profiling** — mean feature values per cluster, business label assignment
8. **Deployment** — Streamlit app for single-customer classification

---

## Algorithm Comparison

| Algorithm               | Silhouette Score | Notes                                    |
|-------------------------|-----------------|------------------------------------------|
| KMeans (K=4)            | ~0.20           | Best overall; chosen for final model     |
| Agglomerative Clustering| ~0.18           | Good hierarchy but slower on large data  |
| Spectral Clustering     | ~0.17           | Sensitive to kernel parameters           |
| DBSCAN                  | ~0.10           | Struggles with varying density clusters  |
| Gaussian Mixture Model  | ~0.19           | Soft assignments; slightly lower score   |

KMeans with K=4 was selected as the final model. The Elbow Method and Silhouette analysis both support this choice, and K=4 produces the most interpretable cluster profiles.

## Cluster Profiles

| Cluster | Label                    | Balance  | Purchases | Cash Advance | Credit Limit | Payment Rate | Key Strategy                        |
|---------|--------------------------|----------|-----------|--------------|--------------|--------------|-------------------------------------|
| 0       | Inactive Users           | Low      | Very Low  | Minimal      | Low          | Low          | Re-engagement campaigns             |
| 1       | Cash-Dependent Customers | High     | Low       | High         | Moderate     | Low          | Personal loan / balance transfer    |
| 2       | Premium Spenders         | Moderate | Very High | Low          | Very High    | High         | Premium rewards / limit upgrade     |
| 3       | Installment Buyers       | Moderate | Moderate  | Low          | Moderate     | Moderate     | EMI / no-cost installment offers    |

### Key Insights

- Approximately 30% of customers fall into the cash advance segment (Cluster 1), indicating a substantial segment with high-cost credit reliance.
- Premium Spenders (Cluster 2) have credit limits roughly 3x higher than other clusters and maintain near-full payment rates, making them the most profitable and lowest-risk segment.

## How to Run Locally

### 1. Clone / navigate to the project directory

```bash
cd credit-card-segmentation
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Run the Jupyter notebook

Open `notebook.ipynb` and run all cells from top to bottom. This will:
- Perform EDA and feature engineering
- Train and compare all five clustering algorithms
- Save `kmeans_model.pkl` and `scaler.pkl` in the same directory

```bash
jupyter notebook notebook.ipynb
```

### 4. Launch the Streamlit app

```bash
streamlit run app.py
```

The app will open at `http://localhost:8501`. Use the sidebar sliders to input a customer's feature values and click **Classify Customer** to see the predicted segment.
